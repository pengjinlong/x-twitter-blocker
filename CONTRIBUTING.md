# Contributing Guide

Thank you for your interest in contributing to **X/Twitter Keyword Blocker**! 🎉

## Ways to Contribute

- 🐛 **Report bugs** — Open an [Issue](../../issues/new)
- 💡 **Suggest features** — Open an [Issue](../../issues/new) with the `enhancement` label
- 🔧 **Submit code** — Fork the repo and open a Pull Request
- 🌐 **Improve translations** — Help us add or improve language support

## Development Setup

1. Clone the repository:
   ```bash
   git clone https://github.com/pengjinlong/x-twitter-blocker.git
   cd x-twitter-blocker
   ```

2. Load the Chrome extension in developer mode (see `README.md` for instructions)

3. Edit `chrome-extension/content.js` or `index.js` and test changes on X/Twitter

## Pull Request Guidelines

- Keep PRs focused on a single feature or fix
- Follow existing code style (no linters required, just keep it clean)
- Write clear commit messages: `Add: feature`, `Fix: bug description`, `Docs: update`
- Update the relevant README sections if your change affects user-facing behavior

## Reporting Bugs

Please include:
- Browser name and version
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if relevant

## Code of Conduct

Be respectful and constructive. We follow the [Contributor Covenant](https://www.contributor-covenant.org/).
